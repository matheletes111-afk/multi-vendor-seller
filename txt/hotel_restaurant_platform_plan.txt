========================================================
  HOTEL & RESTAURANT PLATFORM — FEATURE PLAN
  Multi-Vendor Seller Platform — New Vertical
========================================================

OVERVIEW
--------
A brand-new Hotel & Restaurant vertical is being added to the
existing marketplace platform. This is completely separate from
the current Product Seller and Service Seller flows — it will
not affect any existing functionality.

Customers will be able to:
  - Browse and book hotel rooms (like Booking.com)
  - Order food for delivery (like Swiggy / Zomato)
  - Reserve restaurant tables (with seller confirmation)

Sellers can register under one of three types:
  1. Hotel Only
  2. Restaurant Only
  3. Hotel + Restaurant (hotel with in-house dining)


========================================================
  SELLER REGISTRATION — 3 TYPES
========================================================

TYPE 1 — HOTEL ONLY
  Panel:    /hotel-seller
  Can do:   Add rooms, set availability calendar, manage bookings

TYPE 2 — RESTAURANT ONLY
  Panel:    /restaurant-seller
  Can do:   Add menu items, manage food orders, manage table bookings

TYPE 3 — HOTEL + RESTAURANT (Combined)
  Panel:    /hotel-seller (unified dashboard)
  Can do:   Everything from Type 1 + Type 2 in one place
            (Hotel with its own in-house restaurant)

All three types go through a multi-step onboarding:
  Step 1 — Seller Type Selection
  Step 2 — Business Information (name, address, GST, registration)
  Step 3 — KYC Documents (ID proof front/back + selfie)
  Step 4 — Property Setup (hotel rooms / restaurant profile / both)
  Step 5 — Bank Details (for payouts)
  Step 6 — Agreement & Submit

After submission, Admin reviews and approves the seller account.


========================================================
  SELLER PANELS — WHAT EACH CAN DO
========================================================

HOTEL SELLER DASHBOARD (/hotel-seller)
  - Overview       : Today's bookings, revenue, occupancy rate
  - Rooms          : Add/edit room types, upload photos, set price per night
  - Availability   : Calendar view — block or unblock specific dates per room
  - Bookings       : View all reservations
                     Statuses: Pending → Confirmed → Checked-In → Completed → Cancelled
                     Actions: Confirm / Check-In / Complete / Cancel
  - Restaurant     : Visible only if seller type is "Hotel + Restaurant"
                     (same menu, orders, table sections as restaurant panel)
  - Settings       : Profile, bank details, notifications

RESTAURANT SELLER DASHBOARD (/restaurant-seller)
  - Overview       : Orders today, revenue, pending orders
  - Menu           : Add/edit food categories (Starters, Mains, etc.)
                     Add/edit items: name, price, veg/non-veg, image, availability toggle
  - Orders         : Incoming food orders with full lifecycle management
                     Statuses: Pending → Confirmed → Preparing → Out for Delivery → Delivered
                     Seller can mark each stage
                     OTP is generated when order goes "Out for Delivery"
                     Delivery person enters OTP + uploads proof photo to mark Delivered
  - Tables         : Add/edit tables (table number + seating capacity)
  - Availability   : Block time slots per table per date (for reservation management)
  - Table Bookings : Incoming reservation requests (status = Pending)
                     Seller clicks Accept or Reject
                     Customer is notified automatically
                     Seller receives SMS + Email alert for every new booking request
  - Settings       : Profile, bank details


========================================================
  CUSTOMER EXPERIENCE
========================================================

========================================================
  FOOD DELIVERY — OTP + PROOF FLOW
========================================================

When a food order is dispatched:
  1. Seller marks order "Out for Delivery"
  2. System generates a 4-digit OTP
  3. OTP is shown to the seller (passed to delivery person)
  4. At the customer's doorstep:
       - Delivery person confirms OTP with customer
       - Delivery person uploads a proof photo
  5. System validates OTP → order marked as Delivered
  6. Commission settlement happens automatically

This is the same secure delivery flow used for product orders
on the existing platform.


========================================================
  TABLE BOOKING — NOTIFICATION FLOW
========================================================

  1. Customer submits table booking → status = PENDING
  2. System instantly sends:
       - Email to restaurant seller (with booking details)
       - SMS to seller's registered mobile number
  3. Seller logs into dashboard → Accepts or Rejects the booking
  4. Customer sees updated status (Confirmed / Rejected)

No automatic confirmation — every table booking requires
explicit seller approval.


========================================================
  HOTEL ROOM AVAILABILITY
========================================================

  - Seller uses a calendar view in the dashboard
  - Clicks on dates to block them (e.g. already booked offline, maintenance)
  - Unblocked dates = available for customer booking
  - When customer selects check-in/check-out dates, system checks:
       * No blocked dates in the range for that room
       * No existing confirmed bookings in the range
  - Room pricing is fixed: one price per night per room type
    (e.g. Deluxe Room = $80/night — same rate all year)


========================================================
  COMMISSION MODEL
========================================================

  Hotel Bookings:
    Platform takes a commission on each confirmed room booking.
    Formula: Total Booking Amount × Commission Rate
    Commission rate: global default (set in Admin) or per-seller override

  Food Orders:
    Platform takes a commission on each food order.
    Same formula as hotel bookings.

  Table Bookings:
    No commission — table reservations are free to process.
    (Revenue comes from food orders placed at the restaurant)

========================================================
  CLARIFICATIONS & RECOMMENDATIONS FOR CLIENT
========================================================

1. Onboarding & Registration Form:
   - Do you want to use the EXACT SAME registration form we currently use for Product/Service sellers, or should we create a NEW form specifically for Hotels and Restaurants? (If you want a new form, please give your preferred fields here).
   - For sellers who have BOTH a Hotel and a Restaurant (Type 3), should they fill out one big form at the start, or register as one first and then add the other feature later?

2. Registration vs. Property Setup:
   - To keep registration fast, we recommend a two-phase approach:
     
     PHASE 1: 6-Step Onboarding Flow (Detailed)
     
     Step 1: Selection
     - KEEP: Hotel / Restaurant / Both.
     - REMOVE: Product / Service seller options.

     Step 2: Business Info
     - KEEP: Name, GST, PAN, Address.
     - ADD: Landmark, Manager (POC) Name, and POC Contact.
     - REMOVE: Pickup & Warehouse address fields.

     Step 3: KYC Documents
     - KEEP: ID Proof, Business Proof, and Selfie.
     - ADD: Food License (FSSAI) placeholder for restaurants.

     Step 4: Basic Property Setup
     - FOR HOTELS: Estimate room count, Star Rating (1-5), and Category (Resort/Hotel/etc).
     - FOR RESTAURANTS: Service types (Delivery/Dine-in) and Primary Cuisine.
     - ADD: 
       * Logo & Banner Image (for the property profile).
       * Main Property Photo (Interior/Exterior).
       * GPS Coordinates (Exact location for map navigation).
     - REMOVE: All product-shipping settings like Shipping Partners, Return Policies, and Weight/Dimensions.

     Step 5: Bank Details
     - KEEP: Account Name, Number, Bank, and IFSC (standard for payouts).

     Step 6: Agreement
     - Use specific Hospitality Terms & Conditions instead of standard Product/Service terms.
     
     PHASE 2: Property Setup (Inside the Hotel/Restaurant CRUD)
     - Once registered, the seller fills in their specific details.
   
   RECOMMENDED "SETUP" FIELDS (Moved to CRUD):
   
   FOR HOTELS:
   - Star Rating, Amenities, Check-in/out Policies, Room Types, and GPS Coordinates.

   FOR RESTAURANTS:
   - Cuisine Type, Opening/Closing Hours, Food License (FSSAI), Delivery Radius, and Dietary Tags.

3. KYC Documentation:
   - Do Hotels and Restaurants require any additional legal documents beyond standard ID/Business proof? (e.g., Tourism Board certificates for hotels, Health/Fire safety permits for restaurants).

4. Communication Preferences:
   - For Table Bookings and Food Orders, should we prioritize SMS, WhatsApp, or App Notifications for the seller to ensure fast response times?
