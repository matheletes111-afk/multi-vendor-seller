================================================================================
MOBILE SUBSCRIPTION APIS DOCUMENTATION FOR MOBILE DEVELOPERS
================================================================================

This document details the request payloads, response payloads, headers, and endpoints
for retrieving and switching subscription plans for all 4 types of sellers:
1. Product Sellers (SELLER_PRODUCT)
2. Service Sellers (SELLER_SERVICE)
3. Hotel Sellers (SELLER_HOTEL)
4. Restaurant Sellers (SELLER_RESTAURANT)

--------------------------------------------------------------------------------
1. AUTHENTICATION HEADERS
--------------------------------------------------------------------------------
All mobile subscription APIs require a Bearer token in the Authorization header.

Header Format:
Authorization: Bearer <MOBILE_JWT_TOKEN>

Note: 
- Product and Service sellers use their respective token.
- Hotel and Restaurant sellers must use their respective token.
- Role mismatch will result in a 403 Forbidden error.

--------------------------------------------------------------------------------
2. ENDPOINTS SUMMARY
--------------------------------------------------------------------------------

A. GET CURRENT SUBSCRIPTION & AVAILABLE PLANS
   Retrieve the current active plan, duration, and list of all available plans.

   - PRODUCT & SERVICE SELLER:
     GET /mobileapi/product-seller/subscription

   - HOTEL SELLER:
     GET /mobileapi/hotel-seller/subscription

   - RESTAURANT SELLER:
     GET /mobileapi/restaurant-seller/subscription

B. UPDATE / SWITCH SUBSCRIPTION PLAN
   Switch or upgrade the plan using the plan's name.

   - PRODUCT & SERVICE SELLER:
     POST /mobileapi/product-seller/subscription

   - HOTEL SELLER:
     POST /mobileapi/hotel-seller/subscription

   - RESTAURANT SELLER:
     POST /mobileapi/restaurant-seller/subscription

4. UPDATE / SWITCH SUBSCRIPTION PLAN (POST)
--------------------------------------------------------------------------------

REQUEST PAYLOAD (JSON)
{
  "planId": "plan_id_def", // Preferred: Unique ID of the plan
  "planName": "STANDARD" // Fallback: Enum values: "FREE", "STANDARD", "PREMIUM"
}

Note: Either planId or planName is required.

RESPONSE PAYLOAD (JSON) - Status: 200 OK
{
  "success": true,
  "message": "Subscription updated successfully",
  "subscription": {
    "id": "subscription_id_123",
    "hotelSellerId": "hotel_seller_id_xyz", // or restaurantSellerId / sellerId

    "status": "ACTIVE",

      "type": "HOTEL",
      "displayName": "Standard Hotel Plan",

--------------------------------------------------------------------------------
5. COMMON ERROR RESPONSES
--------------------------------------------------------------------------------

A. Unauthorized / Missing token (401 Unauthorized)
{
  "success": false,
  "error": "Missing or invalid authorization header" // or "Unauthorized or expired token"
}

B. Forbidden / Role mismatch (403 Forbidden)
{
  "success": false,
  "error": "Forbidden: Insufficient permissions"
}

C. Missing fields (400 Bad Request)
{
  "error": "planId or planName is required"
}

D. Plan not found (404 Not Found)
{
  "error": "Plan not found"
}
