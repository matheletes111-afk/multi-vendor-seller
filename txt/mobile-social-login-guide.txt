# Firebase Social Authentication Integration Guide for Mobile Developers

This guide provides details on how to integrate and use native social authentication (Google, Facebook, etc.) with the multi-vendor backend APIs.

---

## 🛠️ Overview of the Authentication Flow

1. **Local Authentication**: The mobile app authenticates the user locally using native Google or Facebook SDKs or Firebase Authentication libraries.
2. **Fetch ID Token**: Once logged in, retrieve the current user's Firebase ID token using:
   - iOS / Android: `currentUser.getIdToken(true)` (this returns the JWT string representing the Firebase ID token).
3. **Send to Backend**: Send a `POST` request to the correct backend route containing the `idToken` and the chosen `provider` ("google" or "facebook").
4. **Backend Verification**: The Next.js server validates the Firebase ID token using the Firebase Admin SDK, extracts user details, checks/creates accounts matching the provider ID, and returns JWT tokens for subsequent authenticated requests.

---

## 🔗 Endpoint Routes

Use the endpoint corresponding to the role of the user logging in:

| Role | Endpoint Route |
| :--- | :--- |
| **Customer** | `/mobileapi/customer/auth/social` |
| **Product Seller** | `/mobileapi/product-seller/auth/social` |
| **Service Seller** | `/mobileapi/service-seller/auth/social` |
| **Restaurant Seller** | `/mobileapi/restaurant-seller/auth/social` |
| **Hotel Seller** | `/mobileapi/hotel-seller/auth/social` |

---

## 📦 Request Payload (JSON)

Perform a `POST` request to the route with the following JSON body structure:

### For Google and Facebook (using Firebase ID Token)
```json
{
  "provider": "google", // or "facebook"
  "idToken": "<FIREBASE_ID_TOKEN>"
}
```

*Note: For backward compatibility, direct logins not using Firebase (e.g. legacy/web oauth flow) are still supported on these endpoints. In those cases, Google uses `idToken` (raw Google ID token) and Facebook uses `accessToken` (raw Facebook access token).*

---

## 📥 Response Structure

### 🟢 Success Response (HTTP 200)
```json
{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": {
      "id": "user-uuid",
      "email": "user@example.com",
      "name": "User Name",
      "role": "CUSTOMER", // or SELLER_PRODUCT, SELLER_SERVICE, etc.
      "phone": null,
      "phoneCountryCode": null,
      "isEmailVerified": true,
      "createdAt": "2026-07-13T12:00:00.000Z",
      "updatedAt": "2026-07-13T12:00:00.000Z"
    },
    "tokens": {
      "accessToken": "<APP_ACCESS_TOKEN>",
      "refreshToken": "<APP_REFRESH_TOKEN>",
      "expiresIn": 86400
    },
    "sessionInfo": {
      "expiresIn": 86400,
      "tokenType": "Bearer"
    }
  }
}
```

### 🔴 Error Response (HTTP 401/403/500)
```json
{
  "success": false,
  "error": "Invalid or expired token" // or other reason
}
```
